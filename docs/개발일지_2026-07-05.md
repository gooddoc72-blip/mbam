# 개발일지 2026-07-05 — Crawler Pro 안정화 (v1.1 → v2.0)

데스크톱 크롤러(`naver_place_crawler`, undetected_chromedriver/selenium)의 연쇄 오류를 잡고 쿠팡 회차 수집을 개편함. 프록시·사업모델 논의는 [프록시_전략.md](프록시_전략.md) 참고.

## 버전별 수정 이력

| 버전 | 문제 | 원인 | 수정 |
|---|---|---|---|
| **v1.1** | 크롤링 초기실행 타임아웃 | `ChromeDriverManager().install()`이 매번 `googlechromelabs.github.io` 조회 → 무한 대기 | 캐시 valid_range=365, 소켓 타임아웃+재시도, 오프라인 폴백 (`resolve_driver_path`) |
| **v1.2** | 엑셀 저장 실패 | OneDrive 백업으로 `~/Desktop` 부재 (`Cannot save into non-existent dir`) | 실제 바탕화면 경로 자동 탐색(OneDrive/바탕 화면 등), 파일 열림 시 새 이름 저장 |
| **v1.3** | 쿠팡 2회차가 1회차부터 재시작 | GUI가 체크포인트 미전달 + 매번 새 엔진 | 회차 루프 내장, page 커서 회차 간 유지, 회차별 엑셀 저장 |
| **v1.4** | IP 변경 `cannot access local variable 'time'` | `ip_changer`의 `import time`이 Mac 분기 안에 있어 Windows에서 지역변수화 | 중복 import 제거 |
| **v1.5** | IP 변경돼도 조용히 실패 | ADB 실패를 print만 하고 IP 변경 검증 안 함 | 기기연결/adb 상태 로그화, 변경 전후 공인IP 비교 검증 |
| **v1.6** | 쿠팡 "페이지당 50건" 부적절 + adb 미설치 | 50 고정이 페이지 중간 자름 / 노트북에 adb 없음 | **페이지 기준 회차**(회차당 N페이지), **adb.exe 동봉** |
| **v1.7** | 초기실행 타임아웃 재발 | wdm은 캐시 있어도 버전매핑 조회로 네트워크 탐 | **캐시 우선**: 로컬 드라이버 있으면 네트워크 완전 생략 |
| **v1.8** | 비행기모드 IP 변경 차단 | `am broadcast AIRPLANE_MODE`가 Android 10+에서 SecurityException | `cmd connectivity airplane-mode`(11+) → `svc data` 토글 → 구형 폴백 |
| **v1.9** | IP 변경 원인 불명 | 테스트 버튼이 로그 미표시 | **IP 변경 자동 진단 도구**(방법별 실측 + WiFi 공유 경고) |
| **v2.0** | 네이버 창 안 열림 / 쿠팡 상세 지연 / 쿠팡 2페이지+ 0개 | ① 네이버만 `use_subprocess=False` ② 상세 페이지로드 타임아웃 없음 ③ 2페이지+ 직접URL이 워밍업 없어 Akamai 차단 | ① 플래그 제거 ② `set_page_load_timeout(30)` ③ 2페이지+ 홈 워밍업 추가 |

## 검증
- `resolve_driver_path`: ChromeDriverManager를 예외 심어도 캐시 경로 반환(네트워크 미호출) 확인.
- IP 변경(v1.8): 실기기에서 `✅ IP 변경 완료: 118.235.13.142 → 39.7.46.9 (방식: airplane-mode(cmd))` 확인. **WiFi 끄고 모바일 데이터 테더링해야 IP 변경됨**.
- 페이지 회차(v1.6/v2.0): 목 하니스로 회차당 60건·페이지 커서 1→3→5→7·중복 0 확인.

## 알려진 남은 이슈 / 후속
- **쿠팡 2페이지+ 차단**: 홈 워밍업은 완화책. 테더링 IP 평판 낮으면 재차단 가능 → **주거용/모바일 프록시 전환이 근본책**.
- **쿠팡 상세수집 속도**: 항목당 수 초~수십 초, 타임아웃으로 상한은 걸었으나 60×N회차는 여전히 오래 걸림.
- **검색창 타이핑 실패**(비치명): 1페이지 진입 시 `headerSearchKeyword` 못 찾고 URL 우회(약 30초 낭비). 셀렉터/대기 최적화 여지.
- IP 변경 방식이 폰별로 다름(방법1 vs 방법2) — v1.9 진단으로 확정 가능.

## 배포
- 매 버전 `Crawler Pro.spec`(PyInstaller) → `setup_script.iss`(Inno Setup) 빌드.
- 배포 파일명 버전 기반: `CrawlerPro_Setup_v{ver}.exe`, 바탕화면 배포.
- 최신: **v2.0** (`CrawlerPro_Setup_v2.0.exe`).
