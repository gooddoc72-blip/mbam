## 1. 순위 추적(블로그·카페 글 순위) 개선

### 1-1. "미검사" 상태 설명 + 순위 변동 그래프/분석 (cff36f7 → 0e96f24)
- **"⏳ 미검사"** = 아직 순위 수집이 한 번도 안 돌아 데이터 없음(history 비어있음). 옆칸 "미노출"은 값이 비어 보이는 것 → **미검사일 땐 순위칸을 `-`로** 표기하도록 변경.
- 순위 항목 **클릭 → 상세**: 요약 타일(현재 통검/탭·최고 통검·검사횟수) + **순위 변동 라인그래프**(인버트 축: 1위=상단, 통검=파랑·탭=주황, 미노출=선 끊김, 포인트 호버) + **매일 순위 분석표**(전일대비 ▲▼).
- 최초 모달로 만들었다가 사용자 요청으로 **행 아래 인라인 펼침/접기**로 변경(`RankDetail` 컴포넌트, `▶` 화살표 회전, 한 번에 하나만 펼침).
- 파일: `mbam-web/app/cafe-rank/page.js`

### 1-2. '노출 키워드'를 제목이 아닌 실제 검색어로 정제 (32cdc62)
- 문제: 매일 자동발행 폴백 글감이 keyword에 **글 제목**(`[EVENT]…`, `[확인필요] … (…)`)을 넣어, 순위추적·제목SEO의 키워드가 검색어가 아니라 제목이 됨(항상 미노출).
- `keyword_seo.clean_search_keyword()` 추가: 앞 대괄호태그·끝 괄호부연·따옴표 제거 → `[확인필요] 부모급여 (0~1세…)` → `부모급여`.
- 적용: `blog_daily_scheduler` 잡 적재 + `blog_schedule` 순위등록 persister. 표 헤더도 타입별(블로그/카페) URL로.

### 1-3. 순위 목록에 발행 글 '제목' 표시 (ac007a9)
- `CafeRankItem.title` 컬럼 추가(+ `cafe_rank_items` 경량 마이그레이션). `register_cafe_rank_if_absent(title=)`.
- 블로그/카페 워크플로 결과에 `title` 포함 → persister가 저장. 프론트: **키워드(파랑) 아래 제목(회색)**, 출처(매일 자동발행)는 더 옅게.

## 2. 제목 SEO — 검색량 기반 '메인+롱테일' 조합 (e0d0e5c)
- 요청: 블로그·카페 제목이 검색량 분석 기반으로 **메인 키워드 + 롱테일 키워드** 조합이 되도록.
- `keyword_seo.suggest_seo_title_keywords()`: 네이버 검색광고 keywordstool로 메인 키워드를 포함하는 확장 검색어 중 **검색량 있으면서 경쟁 낮은** 롱테일 1개 선정. (키/결과 없으면 자동완성 폴백 → 그것도 없으면 메인만, fail-open)
- `orchestrator._generate_content_with_retry`가 재시도 전 1회 계산 → `soul.rewrite_for_blog(long_tail, title_main)` 전달. `_get_blog_prompt`가 롱테일 있으면 '메인 앞배치+롱테일 자연삽입, 32자' 규칙 주입(기본/커스텀 프롬프트 모두).
- 블로그·카페 원고 모두 `rewrite_for_blog` 경로라 공통 적용. 전제: 검색광고 API 키(있음).

## 3. 프록시 IP 풀 + 자동 로테이션 (하이브리드) — ea926e4 / 57ef882 / 48dc53f / 5c48ab3
요청: USB 테더링 외에 **프록시 IP를 여러 개 등록해 자동으로 돌려가며** 사용. 정책은 **하이브리드**(로그인 작업=계정별 고정 IP, 방문/부스트=회전).

- **모델/서비스**: `ProxyServer` 테이블(server/username/password/label/is_active). `services/proxy_pool.py` — 인증 파싱(`user:pass@host:port` 등) + `pick`(hybrid/roundrobin/sticky) + `resolve_proxy`(IP설정 읽어 config) + `to_url`(잡 payload 전송용 왕복). `get_browser_proxy_config`가 인증형 프록시를 `{server,username,password}`로 분리.
- **관리 API/UI**: `proxy_router`(목록/일괄등록/삭제/토글/IP방식설정/연결테스트). `/proxy` 페이지(사이드바 '계정관리' 아래 '프록시 IP') — 프록시 줄단위 등록, IP방식(없음/테더링/프록시)·로테이션(하이브리드/회전/고정) 선택, 연결 테스트.
- **오케스트레이터 연결**: 소통이웃(`execute_engagement_workflow`) + 블로그/카페 발행(`auto_post` trigger에서 계정별 해소 후 `req.proxy`/`accounts[].proxy` 주입 → 로컬·에이전트 동일 적용). 로그엔 인증정보 없이 서버 호스트만.
- IP설정은 사용자별 `AppSetting`(`ip_settings:{user_id}`) KV 저장.
- **미연결(후속)**: 매일 자동발행(blog_daily), 쇼핑/쿠팡 순위수집.

## 4. 소통 & 이웃 — 다계정 + 클라우드 실행 버그 수정
### 4-1. 다계정 (48dc53f)
- `naver_ids` 배열로 계정마다 순차 실행 + **소유권 검증(IDOR)** + DB 비번 복호화 + 계정 사이 랜덤 대기. 프론트: 자동로그인 시 '다계정 동시 진행' 체크박스 목록.

### 4-2. 클라우드 브라우저 실행 오류 → 에이전트 위임 (51c66b7) ★중요 버그
- 증상: `Missing X server`(클라우드 Railway에서 headed 브라우저 실행 실패). 발행과 달리 소통이웃이 **백엔드 프로세스에서 직접 실행**돼 클라우드에서 불가였음.
- 수정: 계정/비번/프록시 해소를 `resolve_engagement_accounts`, 실행 루프를 `run_engagement_loop`로 분리(공용). **클라우드면 `engagement` 잡 적재 → 에이전트가 집 PC에서 실행.** `agent.py`에 `_handle_engagement` 핸들러 추가.
- 프론트: `mode=agent`면 `pollAgentJob`으로 완료까지 폴링·로그 표시.
