## §1. 쇼핑 키워드 분석·정제 모듈
**파일**: `mbam-web/app/shopping/keyword/page.js`, `mbam_nextgen/backend/routers/shopping_router.py`

- **클린 토큰 선택 → 상품명 생성**: 토큰 클릭 선택(전체선택/해제), 선택분만 조립기로 전달.
- **태그 수집**: 상품명에 안 쓰인 연관어를 `#태그`로 수집 + 복사 버튼.
- **태그 ↔ 연관키워드 분리**: 연관키워드를 검색량순 별도 섹션으로 분리, 선택해서 상품명 앞쪽 롱테일로 추가.
- **관련성 필터**: 시드 핵심어를 포함한 검색어만 통과(예: "그릭요거트" → "광주맛집" 제외).
- **태그 = 네이버 등록 키워드 기반**: 상품명 제목 토큰화 노이즈(해썹/시설/밀폐 등) 제거, 검색광고/자동완성 연관어에서만 추출.
- **불용어 다듬기**: `TAG_STOPWORDS`(단위 KG·ml, 채널 정기/배달, 조각 무당 등) + 단위 정규식 제거.
- **상품 해시태그 스크래핑**: 조사 결과 **불가** — 스마트스토어가 자동화 브라우저를 로그인으로 차단(헤드풀·영구프로필·워밍업 모두 실패). 셀러 검색태그는 비공개 메타데이터.

## §2. 카페글 분석 (작성자/카페 권위)
**파일**: `mbam_nextgen/services/seo_analyzer.py`

- **버그**: `analyze_multiple_urls` 카페 분기가 article API JSON에서 제목·본문만 추출하고 작성자/카페 데이터는 누락 → "작성자 정보를 추출하지 못했습니다".
- **수정**: 카페 article API(`apis.naver.com/cafe-web/cafe-articleapi`) JSON에서 닉네임·등급·티어(아이콘URL 파싱)·조회·댓글·작성일·카페명·회원수 추출 + 권위 점수 병합. (Playwright 불필요)
- **좋아요수 수집**: 네이버 통합 LikeIt API(`route-like.naver.com/v1/search/contents?q=CAFE[club_cafeUrl_article]`)로 좋아요 수 추가.

## §3. 카페 댓글/육성 자동화 개편
**파일**: `mbam_nextgen/orchestrator.py`, `mbam_nextgen/backend/routers/cafe_nurture.py`, `mbam-web/app/cafe-auto/page.js`

- **서버오류 분석**: trigger-targeted/add-cafe 엔드포인트는 정상(200), 실제 실패는 ① 백그라운드 워크플로우의 구버전 카페 셀렉터 ② 프론트 stale(새로고침 필요)로 진단.
- **댓글 워크플로우 재작성**: 영구 프로필(기기인증) 로그인 재사용, 신형 ca-fe SPA 대응(다중 폴백 셀렉터 + 실패 시 DOM 덤프), 인코딩 안전 로깅.
- **USB 테더링**: 계정마다 ADB 비행기모드 토글로 IP 로테이션(`use_tethering`) + 현재 IP 로그.
- **댓글 직접입력**: AI 자동생성 → 사용자가 입력한 댓글(여러 줄=무작위 선택). AI 폴백 시 메타/거절성 응답 차단 가드.
- **좋아요 일원화**: 육성 스케줄에서 좋아요 제거 → 댓글 작업에 "댓글+좋아요" 옵션 추가.
- **가입 카페 매핑**: 게시판이름 입력 제거(카페만 매핑, `board_name` 선택사항).
- **작업 유형 라디오 삭제**: 포스팅 탭은 발행 전용으로 정리(댓글은 소통·육성 탭).
- **계정 자동 추가 확인**: `/api/cafe-nurture/accounts`와 `/api/accounts`는 동일 NaverAccount 테이블 → 등록 시 참여 계정 목록에 자동 반영.
