# 플레이스 상위노출 진단 시스템 — N1~N5 연산 설계 (확정본)

> 작성일: 2026-06-07 · 대상: 네이버 스마트플레이스 키워드별 순위 진단/마케팅 의사결정
> 관련 코드: `services/seo_calculator.py`, `backend/routers/place.py`, `services/deep_crawler.py`

---

## 0. 목적 & 핵심 불변식

- **목적:** 키워드별로 1위~이후 업체의 점수를 수치화하여, **내 매장이 무엇을 마케팅해야 순위가 오르는지** 처방한다.
- **핵심 불변식:**
  1. N5는 **순위를 입력으로 받지 않는다**(순환 금지). 실데이터로만 산출한다.
  2. 올바르면 **N5_eff 내림차순 = 실제 네이버 순위**와 거의 일치해야 한다. → 1위는 N5_eff 최상위가 정상.
  3. **N5_eff − N5_organic(=부스트·조작 제거)** 의 갭이 곧 마케팅 인사이트(취약 1위, 공략 가능).

## 1. 점수 체계 개요

| 지표 | 의미 | 주요 입력 | 성격 |
|---|---|---|---|
| **N1** | 플레이스 정적 품질 (키워드 적합도 + 정보 전반) | 소개글·업체명·카테고리 키워드, 정보완결성, 사진, 예약/주문기능, 영업상태, 리뷰최신성, 답글, 소식 | 정적 |
| **N2** | 내부 인기 / Engagement | 영수증·방문자·**예약자** 리뷰(+증감), 저장수(버킷, 약가중), 리뷰조회수, 키워드유입율(추정) | 동적 |
| **N3** | 외부 확산 | 블로그 리뷰(+증감), 블로그 조회수, C-Rank | 동적 |
| **N4** | 어뷰징 필터 (곱셈 0~1) | 순위–펀더멘털 잔차, 순위 급등속도, 순위 변동성 | 패널티 |
| **N5** | 종합 총점 | 위 합성 | 결과 |

```
N5 = ( w1·N1 + w2·N2 + w3·N3 ) × N4        (w1+w2+w3 = 1.0)
```
- N4는 **가산이 아니라 곱셈** (어뷰징 시 N5 붕괴 = 네이버 의도).
- 시작 가중치(이후 캘리브레이션): **w1=0.30, w2=0.40, w3=0.30**.

## 2. 공통 전처리 (모든 누적/증감 지표에 적용)

```
1) 로그 압축:   X' = ln(X + 1)
2) 군 Min-Max:  X̃ = (X'-X'_min) / (X'_max-X'_min)     # "같은 키워드 경쟁사" 군 내부, 분모0이면 0.5
3) 시간 감쇄:   X_decay = X · e^(-λ·t),  λ = ln(2)/7    # t=실제 경과일 (반감기 7일)
```
- 핵심: **고정 분모 정규화 금지.** 군 내부 Min-Max라야 화면의 "N1 전부 0.369 / N3 순위복제" 압축 문제가 해소됨.
- 구현은 **2-pass**: ① 전 경쟁사 raw 수집 → 군 min/max 산출 → ② 정규화·점수 계산.

## 3. N1 — 플레이스 정적 품질 (키워드 적합도 + 정보 전반)

> **정의 확정:** 플레이스 리스팅 자체에 내재된 품질·적합도 전부를 N1에 통합한다(리뷰 계열은 제외 → N2/N3).

```
N1 = w1a·키워드적합 + w1b·정보완결성 + w1c·콘텐츠품질 − 스팸패널티
```

| 구성 | 산출 |
|---|---|
| 키워드적합 (w1a≈0.45) | 카테고리 일치도 + 업체명 키워드 포함 + 소개글 키워드 자연밀도(Kiwi, 통상 1~3%) |
| 정보완결성 (w1b≈0.25) | 영업시간·전화·메뉴·주소·예약/톡톡/스마트주문 기능 충실도 |
| 콘텐츠품질 (w1c≈0.30) | 사진 수/대표사진, 리뷰 최신성(마지막 리뷰일), 사장님 답글률, 소식·쿠폰 발행 활동성 |
| 스팸패널티 | 소개글 키워드 도배(스터핑) 시 감점 (네이버 정책) |

- **영업상태 필터:** 폐업/임시휴업은 점수화 제외(사전 필터).
- 데이터: 소개글·카테고리·기능·사진은 **상세 페이지(entryIframe)** 필요 → 수집 비용 큼. **절충: 타겟 + Top3(또는 Top30)만 상세 수집**, 나머지는 업체명·카테고리로 근사.

## 4. N2 — 내부 인기 / Engagement

```
N2 = w2a·I_f + w2b·Ṽ_r + w2c·R̃_eng + w2d·S̃_save
```

| 구성 | 산출 | 비고 |
|---|---|---|
| I_f 유입율(CTR) (w2a) | 노출 대비 상세진입 **추정 프록시** | ⚠ 네이버 내부값 → 추정, 보수적 가중 |
| Ṽ_r 리뷰조회수 (w2b) | 로그+군정규화 | 수집 가능 시 |
| R̃_eng 리뷰 engagement (w2c) | 영수증·방문자·**예약자 리뷰** 누적 + **증감(7일)** | 증감 고가중 (실시간성) |
| S̃_save 저장수 (w2d, 약가중) | **버킷 중앙값** log+군정규화 | 정확값 없음 → 증감 미사용 |

- **리뷰 계열(영수증·방문자·예약자)은 N1이 아니라 여기.**
- **저장수:** 네이버 미공개·버킷만 → 체급 신호로만 약하게, **증감은 쓰지 않음**(§6 참조).

---

## 5. N3 — 외부 확산 (블로그/C-Rank)

```
N3 = w3a·R̃_b + w3b·Ṽ_b + w3c·C_rank        (w3a≈0.45, w3b≈0.25, w3c≈0.30)
```
| 구성 | 산출 |
|---|---|
| R̃_b | 블로그 리뷰 수 + **증감** (log+군정규화) |
| Ṽ_b | 블로그 실조회수 (없으면 R̃_b로 대체, w3b↓) |
| C_rank | 상위 블로거 C-Rank 평균(전문성/일관성) — 미수집 시 0.5 중립 |

- 현재 코드의 `v_b = blog*100`(가짜 조회수)은 제거 — R̃_b와 신호 중복.

## 6. N4 — 어뷰징 필터 (유입·저장 조작 "역추적")

> **딜레마:** 조작자가 펌핑하는 **유입율·저장수가 바로 네이버 미공개 변수.** 직접 측정 불가.
> **해법:** 조작의 *지문* = "보이는 노력(리뷰) 없이 순위만 급등" → **순위(관측 가능)로 역추적.**

```
N4 = clamp(1 − M, floor=0.1, 1)
M  = wA·잔차⁺ + wB·급등속도 + wC·순위변동성       (각 군 정규화)
```

| 신호 | 정의 | 잡는 것 |
|---|---|---|
| **A. 순위–펀더멘털 잔차** | 실제 순위상승 − f(영수증증감, 블로그증감, 예약증감, N1변화) | 유입·저장 조작 |
| B. 급등 속도 | 단기 순위 상승폭이 군 대비 비정상 | 자연=완만, 조작=급등 |
| C. 순위 변동성(롤백) | 순위 시계열 분산 (급등 후 급락 반복) | 조작 on/off·네이버 롤링 |

- 메인 = **A(잔차)**. 초기엔 규칙기반(리뷰 1건당 기대 순위상승 계수) → 데이터 축적 후 회귀.
- **⚠ 신규오픈 면제:** `is_new & t_open<window`면 A·B 제외(신규부스트도 동일 패턴).
- 출력은 **판정이 아니라 "의심도 M + 신뢰구간"** (정상 급성장과 구분 한계).

## 7. 신규오픈 부스트 모델 B(t)

> 네이버는 신규오픈에 가점 → **방문자·영수증·블로그·예약 리뷰 값(및 증감)을 증폭**시켜 순위 상승.
> **가산(+) ❌ → 시간감쇄 곱셈(×) ⭕.**

```
B(t) = 1 + β · e^(−λ_b · t_open)        # t_open=개업 후 경과일
적용: N2/N3의 리뷰·방문·예약 지표 ×= B(t)
```
- β(초기 증폭률), λ_b, window는 **신규오픈 업체들의 경과일 대비 순위하락 곡선**으로 캘리브레이션.
- t_open: 개업일 노출 시 직접 사용, 없으면 **"새로오픈" 뱃지 소멸 시점**으로 역산.

## 8. N5 — 종합 (eff vs organic)

| 점수 | 정의 | 용도 |
|---|---|---|
| **N5_eff** | 부스트·현 상태 반영 = `(w1N1+w2N2+w3N3)×N4` (N2/N3에 B(t) 적용) | **실제 순위와 일치** (불변식 검증) |
| **N5_organic** | 부스트(B=1) + 조작(M=0) 제거 | **지속 가능한 실력 순위** |

- 마케팅 처방의 기준은 **N5_organic**. "조작/부스트 멈추면 갈 순위".

---

## 9. 키워드유형·카테고리별 보정

- **키워드 유형 분류:**
  - 지역+업종("광안리 맛집") → N1 적합도 정상 적용.
  - 브랜드("스타벅스") → N1 적합도 비중↓(상호 일치 자명).
  - 일반어("회") → 카테고리 일치 비중↑.
- **카테고리별 baseline:** 맛집/병원/학원은 리뷰 자연성장률·저장 패턴 상이 → **N4 어뷰징 임계를 카테고리별**로(오탐 방지). 군 정규화는 "같은 키워드 경쟁사" 내부에서.

## 10. 마케팅 의사결정 출력 (업체별)

```
격차 = { N1:Δ, N2:Δ, N3:Δ } = (1위 또는 Top3평균) − 내매장
최대격차 지표 → 1순위 액션
 - N1 최대 → 소개글 키워드/정보완결/사진·예약 보강
 - N2 최대 → 영수증·예약 리뷰 속도↑, 유입·저장 유도, 대표사진 개선
 - N3 최대 → C-Rank 높은 블로거 협업

취약 1위 탐지: 실제1위 N5_organic < Top10 중앙값  →  "공략 가능" 배지
추월 공식: t_remain(부스트 잔여) 안에 내 N5_organic ≥ 그 업체 N5_organic
          → "주당 영수증 +X·블로그 +Y 유지 시 부스트 종료와 동시 추월"
```

## 11. 검증 루프 & 캘리브레이션

- **검증 지표:** `Spearman(N5_eff 순서, 실제 순위) ≥ 0.8`. 미만이면 "모델이 현실과 괴리" 경고 + 가중치 재보정 트리거.
- **가중치 학습:** Ridge 회귀 `L = Σ(rank_target − N5)² + α·Σw²` (단일 지표 폭주 방지). 초기엔 §1 고정값.

## 12. 데이터 소스 & 수집 (확정 필요 ⚠)

| 항목 | N매핑 | 수집 가능성 | 비고 |

| 영수증·방문자 리뷰(+증감) | N2 | ✅ 일별 diff | |
| **예약자 리뷰** | N2 | ❓ **확인 필요** | |
| **저장수** | N2(약) | △ **버킷만** | 증감 미사용 확정 |
| **리뷰 조회수** | N2 | ❓ **확인 필요** | |

| 블로그 조회수/C-Rank | N3 | ❓/△ | 미수집 시 중립 |

- **Cold start:** 과거 스냅샷 없는 키워드/신규 추적은 초기엔 **N1·N2 정적 점수만**, 증감·잔차·변동성은 "데이터 축적 중" 표기.

## 13. 알려진 제약

- **🔴 검색 좌표 = 서버 기준(고정 좌표 아님):** 네이버 순위는 검색자 위치로 개인화됨. 현재는 서버 위치 기준 순위만 관측.
  - 영향: 같은 키워드라도 "지역적 진짜 순위"와 다를 수 있고, 위치가 흔들리면 N4 잔차/변동성이 오염됨.
  - **완화책(단기):** 수집 위치를 **항상 동일 서버/IP로 고정** → 절대 순위는 아니어도 **일별 비교는 일관**되게 유지.
  - **개선(중기):** 키워드별 **기준 좌표(매장 인근 1점) 고정 검색**으로 전환 + 거리 요인을 모델 상수로 분리.
- CTR·저장 증감·블로그 조회수·C-Rank는 정확 수집 한계 → 추정/중립 처리, 가중치 보수적.

## 15. 구현 로드맵

1. **(즉시 효과)** `seo_calculator`: N5 곱셈공식 + rank 제거 + N4 곱셈 + N1 정적품질화.
2. `place.py`: **2-pass 군 정규화** 리팩터, 실제 리뷰/증감 사용, random·추정·타임스탬프 버그 제거.
3. **신규오픈 B(t) 곱셈 부스트** + N5_eff/N5_organic 분리.
4. **N4 잔차/속도/변동성** (순위 시계열 기반) + 신규오픈 면제.
5. 키워드유형·카테고리 보정, Spearman 검증 루프, 마케팅 처방 출력.
6. (추후) CTR·조회수 추정 모듈, Ridge 가중치 학습.

> ①②만 적용해도 화면의 "N1 전부 동일·N3 순위복제" 즉시 해소.
