## 2. 크롤링용 vs 재판매용 차이
- **크롤링용** = 내가 사서 **직접 수집**에 씀. 중요: 한국 IP 품질·차단 저항·속도·GB 단가.
- **재판매용** = 도매로 사서 **고객에게 되팖**. 중요: 화이트라벨, 서브계정/대역폭 재분배, 리셀러 API, 마진, 재판매 허용 약관.
- Bright Data/Oxylabs = 품질 최고지만 **재판매 사실상 금지**. 재판매는 Rayobyte/Infatica 계열.

---

## 3. 프록시가 "필수"인가? (기능별로 다름)
| 기능 | 프록시 필수? | 이유 | 프록시 성격 |
|---|---|---|---|
| **네이버 다계정 블로그 포스팅** | **필수** | 같은 IP면 계정 묶여 정지/저품질 | 계정별 **sticky 주거용** |
| **쿠팡 순위분석** | 필수 아님 | 계정 없음 → IP 분리 불필요, 차단회피만 | **로테이팅**(막힐 때만) |

- **핵심**: 프록시 투자는 사실상 **"다계정 블로그"** 때문. 쿠팡 순위분석만이면 테더링으로도 버팀.
- 플랫폼 민감도: **네이버 > 티스토리 > 블로그스팟**.

---

## 4. 테더링 vs 프록시 (다계정 포스팅 IP 방식)
| | 테더링(폰 USB) | 프록시(주거용) |
|---|---|---|
| 비용 | 무료(고객 폰) | 유료(GB) |
| 동시 IP | **1개**(순차) | 여러 개(병렬) |
| 규모 | 소규모·순차 | 대규모·병렬 |
| 설치 난이도 | 높음(폰+USB+디버깅, WiFi off 필수) | 낮음(주소 입력) |
| 계정 육성 안전성 | 보통(IP 계속 변함) | **높음**(계정별 고정) |

- 테더링은 **한 번에 IP 1개** → 다계정 병렬 불가, 순차만.
- 육성엔 **로테이션보다 계정별 sticky IP**가 안전.

## 5. 재판매는 필수인가? → **아니요**
- 갈림길: **서버에서 대신 실행(server-side)** vs **고객 PC에서 실행(설치형)**.
- **설치형(고객 PC)**: 고객이 자기 테더링 or **자기 프록시 입력(BYOP)** → **재판매 불필요.**
- "프록시가 필요하다" ≠ "프록시를 재판매한다". (필요하면 내 원가로 사서 쓰면 됨)

---

## 6. 최종 사업모델 방향 (2026-07-05 논의 결론)
**설치형(구매형) 데스크톱 앱 + AI는 서버 과금(구독)** 하이브리드로 가닥.

```
[설치형 데스크톱 앱]  ← 고객 PC에서 실행
  ├─ IP: 테더링(무료) or 프록시 BYOP  ← 고객 선택, 재판매 불필요
  ├─ 블로그 로그인·포스팅: 고객 계정, 고객 PC  ← 프라이버시·리스크 회피
  └─ AI 글생성/분석: 우리 서버 API 호출        ← 과금·구독 근거
```
- 장점: 재판매 불필요, 테더링/프록시 선택 자연스러움, 계정 프라이버시, 서버 브라우저 비용 회피.
- 대가: **배포·업데이트·지원 부담**(자동 업데이트·설치 가이드로 완화 필요), 라이선스(HWID), OS별 빌드.
- 구독료 원가 = **AI + (사용 시)프록시 + 서버 분담**. "AI 비용으로만"은 프록시/서버 누락 위험 → 사용량 캡 필요.
- 미확정: AI 처리를 (a) 우리 서버 API(과금 가능) vs (b) 고객 자기 키(과금 불가) — **(a) 권장**.

## 7. 실측 방법
- 도구: `naver_place_crawler/proxy_test.py` (requests 기반 1차 스크리닝).
- 절차: 업체 가입 → KR 게이트웨이/아이디/비번 발급 → `PROVIDERS`에 입력 + `enabled=True` → `python proxy_test.py`.
- 측정: 공인IP/국가/ISP, 네이버·쿠팡 성공률, 응답시간, 데이터(MB)→비용 환산.
- 판정: 쿠팡 성공률 70%+면 실사용 가능선. 고유IP 수 ≈ 반복횟수면 로테이션 정상.
- 통과 업체 연동: uc 크롤러엔 `selenium-wire`, MBAM NextGen(Playwright)엔 `SHOPPING_PROXY` 환경변수.

## 8. MBAM NextGen 프록시 배관 현황 (참고)
- `ProxyManager.get_browser_proxy_config()` 있음(`{"server": url}`만 반환 → **인증 프록시 user/pass 분리 처리 필요**).
- `SHOPPING_PROXY` 환경변수로 네이버 쇼핑 흐름에만 연결됨. **쿠팡 라우터는 프록시 미연결.**
- 테더링 `rotate_ip()`가 구버전 `am broadcast`(Android 10+ 차단) 사용 → 최신 방식 교체 필요(데스크톱 v1.8에서 해결한 방식 이식).
